
# structure-job
调度中心

## 项目简介

这是一个基于 XXL-JOB 的分布式任务调度中心，用于管理和调度各种定时任务。该项目提供了可视化的任务管理界面，支持任务的动态配置、调度和监控。

## 功能特性

- 任务管理：支持任务的创建、修改、删除和暂停
- 调度管理：支持 CRON 表达式配置任务执行时间
- 执行器管理：统一管理任务执行器
- 日志监控：提供任务执行日志查看和分析
- 失败重试：支持任务失败自动重试机制
- 分片广播：支持任务分片执行，提高执行效率
- 权限管理：提供用户权限控制和访问安全

## 技术架构

- 核心框架：XXL-JOB
- 前端技术：Vue.js + Element UI
- 后端技术：Spring Boot + MyBatis
- 数据库：MySQL
- 注册中心：支持 Nacos、Zookeeper 等

## 环境要求

- JDK 1.8+
- MySQL 5.7+
- Maven 3.6+
- Node.js 12+ (前端项目构建)

## 使用指南

### 添加新任务

1. 登录调度中心管理界面
2. 进入"任务管理"页面
3. 点击"新增"按钮
4. 配置任务信息：
   - 任务描述：填写任务名称和描述
   - 调度时间：配置 CRON 表达式
   - 任务模式：选择 BEAN模式 或 GLUE模式
   - 任务参数：填写任务执行参数
5. 保存并启动任务
### 任务监控

- 实时调度日志查看
- 任务执行成功率统计
- 执行器在线状态监控
- 任务失败告警通知

支持动态负载均衡，多个执行器实例可以自动分担负载：

- ROUND：轮询方式
- FAILOVER：故障转移
- BUSYOVER：忙碌转移
- SHARDING_BROADCAST：分片广播

## 常见问题

### 1. 任务没有执行

- 检查执行器是否在线
- 检查 CRON 表达式是否正确
- 查看任务日志排查错误

### 2. 调度中心无法访问

- 检查端口是否被占用
- 检查网络连接是否正常
- 检查防火墙设置
